# 用户模块 Brainstorm

> Stage 0 | 2026-04-26 | 下一步：design

## 想做什么、为什么

用户想补齐类似知乎/小红书/抖音的用户模块功能。当前 clx-user 服务是空壳，用户无法：
- 查看自己的个人主页
- 查看别人的个人主页
- 关注其他用户
- 收藏帖子
- 编辑个人资料

这是社区产品的核心功能缺失，需要一口气补齐。

## 考虑过的方向

### 方向 A：一口气全做（选定）
- 范围：L1 用户资料 + L2 个人主页 + L3 社交关系
- 价值：一次性补齐，体验完整
- 代价：工作量大，需仔细拆分 design
- 结论：**选定**

### 方向 B：分三个 Feature 递进
- 范围：先做资料服务，再做主页，最后做社交
- 价值：每步独立验收，风险低
- 代价：要开三份 design，走三轮流程
- 结论：否决（用户希望一次性完成）

## 选定方向与遗留问题

**核心行为**：
- 用户可查看/编辑个人资料（头像、昵称、签名）
- 用户有个人主页（三栏布局，和首页统一）
- 用户可关注/取关其他用户
- 用户可收藏/取消收藏帖子
- 关注/粉丝/获赞三数展示

**明确不做**：
- 私信功能（已有 message 服务，前端对接另开 feature）
- 黑名单/屏蔽
- 用户等级/勋章系统
- 推荐算法

**设计决策已拍板**：
| 项目 | 选择 |
|------|------|
| 布局风格 | 三栏延续（和首页统一） |
| 关注/粉丝展示 | 三数展示（关注/粉丝/获赞） |
| 收藏夹层级 | 单级收藏（无分类） |

**遗留给 design 的问题**：
1. 关注/粉丝数的实时更新策略（同步 vs 异步）
2. 获赞数统计来源（帖子点赞数累加？）
3. 个人主页 URL 设计（/user/{id} 还是 /u/{username}？）
4. 收藏表的归属（放 clx-post 还是 clx-user？）
