# 刷题系统 Brainstorm

> Stage 0 | 2026-04-25 | 下一步：design

## 想做什么、为什么

用户想在 CLX 社区平台增加刷题功能，参考 jc-club 项目的设计，但功能要更复杂。

最终目标是打造一个**大而全的刷题+考试系统**，覆盖四个场景：
- A) 社区自娱：自愿参加，积分/徽章奖励
- B) 企业培训：强制参加，成绩归档
- C) 教育机构：课程绑定，学分统计
- D) 模拟面试：随机出题，模拟面试场景

探索中发现用户要求的功能复杂度很高（编程题+重度防作弊+智能AI推荐+深度社区整合），一次性做完不现实，因此决定**分期开发**。

## 考虑过的方向

### 方向 A：一次性全做
- 设计完整架构，一次性开发全部功能
- 结论：否决。复杂度过高，开发周期太长，风险太大

### 方向 B：先做考试系统
- 先把考试场次、限时、防作弊做出来
- 结论：否决。考试系统依赖题库，没有题库就无法考试，逻辑上刷题应该先行

### 方向 C：先做题库+基础刷题（选定）
- 第一期聚焦题库管理 + 基础刷题
- 后续分期扩展编程题、考试系统、AI推荐、防作弊、社区整合
- 结论：选定。这是最合理的分期路径，每期都有独立闭环

## 选定方向与遗留问题

**第一期范围**：
- 题库管理：单选/多选/判断/简答四种题型，分类/标签/难度/解析
- 基础刷题：自由练习、专项练习、错题本
- 暂不做：编程题（在线判题）、AI智能推荐、考试系统、防作弊、社区深度整合

**后续分期规划**：
| 分期 | 内容 |
|------|------|
| 第二期 | 编程题（在线判题引擎 Judge0/Sandbox） |
| 第三期 | 考试系统基础版（场次、限时、成绩、排名、证书） |
| 第四期 | 重度防作弊（摄像头监控、屏幕录制、AI行为分析） |
| 第五期 | AI智能推荐（知识图谱、学习路径规划、自适应难度、预测薄弱点） |
| 第六期 | 社区深度整合 + 积分系统 + 徽章成就 |

**遗留给 design 的问题**：
1. 模块划分：新建 `clx-quiz` 一个模块，还是拆成 `clx-subject` + `clx-practice` 两个模块？
2. 题目类型处理：策略模式处理四种题型（单选/多选/判断/简答）？
3. 数据库表设计：题目主表、分类表、标签表、练习记录表、错题本表的具体字段
4. 前端页面规划：题库首页、题目详情页、答题页面、错题本页面、分类/标签筛选页
5. 与现有模块整合：认证复用 clx-auth，积分系统预留接口
6. 后续扩展架构预留：编程题接口预留、考试系统数据结构预留
