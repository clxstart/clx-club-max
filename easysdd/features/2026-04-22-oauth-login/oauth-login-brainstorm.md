# 第三方登录与手机号登录 - 头脑风暴

## 当前现状

**已有功能**：
- 用户名+密码登录 ✅
- 邮箱+验证码注册/找回密码 ✅
- JWT Token 认证体系 ✅

**用户痛点**：
1. 每次都要记密码，容易忘
2. 注册流程太麻烦，要填很多信息
3. 年轻人更喜欢扫码登录

---

## 方案发散

### 方案A：GitHub OAuth（开发者首选）

**优点**：
- 开发者用户最熟悉
- 文档完善，接入简单
- 不需要审核，立即能用

**缺点**：
- 非开发者用户没有GitHub账号
- 国内访问可能不稳定

**实现复杂度**：⭐⭐（简单）

---

### 方案B：微信登录（国内必备）

**两种方式**：

| 方式 | 适用场景 | 复杂度 |
|------|---------|--------|
| 微信网页授权 | PC网站 | ⭐⭐⭐ |
| 微信扫码登录 | PC网站 | ⭐⭐⭐⭐ |

**优点**：
- 国内用户几乎人人都有
- 信任度高

**缺点**：
- 需要企业资质认证
- 个人开发者难申请
- 审核周期长

---

### 方案C：QQ登录（年轻用户）

**优点**：
- 年轻人常用
- 个人开发者可以申请
- 审核相对简单

**缺点**：
- 文档较旧
- 用户体验不如微信

**实现复杂度**：⭐⭐⭐

---

### 方案D：手机号登录（最通用）

**两种方式**：

| 方式 | 说明 | 成本 |
|------|------|------|
| 短信验证码 | 真实短信 | 0.05元/条 |
| 一键登录 | 运营商认证 | 0.1元/次 |

**优点**：
- 所有人都有手机号
- 不需要记密码
- 用户体验最好

**缺点**：
- 短信费用
- 需要对接服务商

**推荐服务商**：
- 阿里云短信
- 腾讯云短信
- 网易云信

---

### 方案E：Gitee登录（GitHub国内替代）

**优点**：
- 国内访问快
- 开发者熟悉
- 支持OAuth

**缺点**：
- 用户量不如GitHub

---

## 技术实现思路

### 统一登录流程设计

```
用户点击登录按钮
    │
    ├─→ 用户名/密码登录（已有）
    ├─→ 手机号+验证码登录
    ├─→ GitHub登录
    ├─→ QQ登录
    └─→ 微信登录
            │
            ▼
    第三方授权成功
            │
            ▼
    后端处理回调
    ├─→ 用code换token
    ├─→ 获取用户信息
    ├─→ 查找是否已绑定
    │       ├─→ 已绑定 → 直接登录
    │       └─→ 未绑定 → 创建新用户或绑定已有账号
    │
    └─→ 返回JWT Token
```

### 数据库设计

```sql
-- 社交账号绑定表
CREATE TABLE sys_social_bind (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '本地用户ID',
    
    -- 第三方平台信息
    social_type VARCHAR(20) NOT NULL COMMENT '平台类型:github/qq/wechat/phone',
    social_id VARCHAR(100) NOT NULL COMMENT '第三方唯一标识',
    
    -- 用户信息（冗余存储，方便展示）
    social_name VARCHAR(100) COMMENT '第三方昵称',
    social_avatar VARCHAR(500) COMMENT '第三方头像URL',
    
    -- Token信息（可选，用于后续操作）
    access_token VARCHAR(500) COMMENT '访问令牌',
    refresh_token VARCHAR(500) COMMENT '刷新令牌',
    token_expire_time DATETIME COMMENT '令牌过期时间',
    
    -- 元数据
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 唯一约束：一个平台的同一个账号只能绑定一次
    UNIQUE KEY uk_social_bind (social_type, social_id),
    
    -- 索引
    KEY idx_user_id (user_id)
) ENGINE=InnoDB COMMENT='第三方社交账号绑定表';
```

## 待决策问题

### 问题1：先做哪个平台？

**选项**：
- A. GitHub（开发者友好，立即能用）
- B. 手机号（最通用，但需要费用）
- C. QQ（国内用户多，个人可申请）
- D. 全部一起做

**我的建议**：GitHub → 手机号 → QQ → 微信

### 问题2：首次第三方登录如何处理？

**选项A：自动创建用户**
- 优点：流程简单，一键登录
- 缺点：可能产生大量僵尸账号

**选项B：引导绑定已有账号**
- 优点：账号归一，方便管理
- 缺点：流程复杂，多一步操作

**选项C：让用户选择**
- 优点：灵活
- 缺点：开发复杂

### 问题3：是否支持绑定多个第三方账号？

**建议**：支持！
- 一个用户可以绑定 GitHub + QQ + 微信
- 任意一种方式都能登录
- 方便用户，提升体验

### 问题4：手机号登录是否支持注册？

**建议**：支持！
- 首次手机号登录 = 自动注册
- 生成随机用户名（如：user_123456）
- 用户后续可修改用户名
